# pcr-rainbow-type-power-optimizer

根据角色的基础面板、当前最优 EX 装备配置，计算满词条彩武（20.68% 攻击）的极限战力，并通过贪心算法模拟多轮彩武分配，自动选出能使全队总战力最大化的彩武类型（弓、拳套、剑、杖等），同时输出完整的武器转移链条。  
当前基于 Lv.344 满练度环境进行数值拟合。

## 特性

- 任务一：为所有角色计算换装满词条彩武后的理论最高战力，并按净提升排序。
- 任务二/三：给定副词条百分比（如 8.8%）和轮次，贪心模拟最优彩武类型分配，同时实现旧武器向下转移，最大化全队总战力增益。
- 所有计算严格按照战力公式（等级、技能、专武、6星面板系数）。
- 自动生成带时间戳的详细报告，记录每轮分配细节。
- 数据与代码分离，`data/` 文件夹存放所有输入文件。

## 数据文件格式说明

### 1. `base_panel_stats.txt` —— 角色原始面板（不穿EX装备）

每行一个角色，格式：  
`角色名 战力:数字 物攻:数字 魔攻:数字 物防:数字 …`  
示例：

    猫拳 战力:96879 物攻:60353 魔攻:0 物防:1014 魔防:863 物爆:2542 魔爆:0 …

### 2. `optimal_panel_stats.txt` —— 当前最优配置下的面板

格式同上，但为穿戴了最优 EX 装备后的数值与战力，程序仅读取其中的战力字段用于对比。

### 3. `best_ex_equip_setups.txt` —— 各角色最优 EX 配装方案

格式：  
`角色名 装备 武器描述, 衣服描述, 饰品描述`

其中装备描述支持彩武专属格式：  
`彩-霸瞳神杖★0[魔攻x4.68%](魔攻x14.40%)`  
括号外为主词条，括号内为副词条，多个词条用 `/` 分隔。金装/粉装直接给出百分比加成。

### 4. `normal_ex_equip_info.txt` —— 普通EX装备类型表

每行一个装备：`装备名 类型`  
示例：

    粉-天国之拳 拳
    金-白马战弓 弓
    粉-元素之剑 单手剑

### 5. `rainbow_weapon_info.txt` —— 彩武基本信息

每行描述一把彩武，包含主词条和类型标识。  
示例：

    彩-霸瞳神弓 天之麻迦古弓 彩弓 主词条 物攻4.68% 副词条4%物攻*4

### 6. `current_level.txt` —— 当前角色等级

仅包含一个三位数（如 `344`），程序用它来计算技能战力、专武战力、6星战力等固定部分。

## 使用说明

### 环境要求

- Python 3.6 及以上（无需第三方库）
- 确保 `data/` 文件夹与脚本在同一目录，且包含上述 6 个文件

### 运行步骤

1. 克隆或下载本仓库到本地。
2. 检查 `data/current_level.txt` 中的等级是否正确，如与游戏不符请修改。
3. 打开终端/命令行，进入仓库根目录，执行：

       python pcr_optimizer.py

4. 根据提示输入彩装副词条百分比（例如 `8.8` 表示 8.8%），然后输入模拟轮次（例如 `5`）。
5. 程序自动完成计算，并在控制台输出以下内容：
   - 满词条彩武（20.68% 攻击）净提升前 20 名
   - 每轮贪心分配选择的彩武类型、获得新彩武的角色、旧武器转移链及战力变化
   - 全队总战力净增长

6. 程序会在同目录生成报告文件 `report_20260506_143022.txt`（时间戳为当前时间），包含完整结果。

### 输入参数说明

| 参数 | 含义 | 典型值 |
|------|------|--------|
| 彩装副词条百分比 | 新彩武副词条提供的攻击加成总和（不含主词条4.68%） | 8.8（代表 4 个 2.2% 或 4%+4%+0.8% 组合） |
| 模拟轮次 | 要分配的新彩武数量 | 5 |

## 战力计算原理

本工具遵循《公主连结》角色战力公式，综合了以下组成部分：

总战力 = 基础战力(150) + 面板战力贡献 + 技能战力 + 专武战力 + 二专战力 + 6星战力

- 面板战力：所有面板属性 × 对应系数之和（系数参考：物攻/魔攻 ×1.0，HP×0.1，双防×4.5，双爆×0.5 …）。
- 技能战力：`等级 × 40`
- 专武/二专战力：`等级 × 2 + 100`
- 6星战力：`等级 × 5 + 2000`

当前脚本假设所有角色均已开启专武、二专与 6 星，故使用相同的固定战力附加值。如需特殊处理未满练度角色，可修改代码中的 `FIXED_POWER_SUM` 计算逻辑。

满词条彩武指主词条（4.68% 攻击）+ 四个最大副词条（4%×4），共 20.68% 攻击加成。  
贪心算法模拟时，每一轮遍历全部 10 种彩武类型，找出能使当前全队总战力提升最大的类型，并模拟旧武器沿攻击力队列向下转移，直至无正提升为止。

## 进阶修改

- 更改等级：直接修改 `data/current_level.txt` 中的数字即可，程序会自动重算固定战力部分。
- 更新角色数据：按格式更新 `base_panel_stats.txt` 等文件，脚本无需改动。
- 模拟规则调整：贪心策略、武器转移条件等可在 `simulate_greedy` 函数中修改。
